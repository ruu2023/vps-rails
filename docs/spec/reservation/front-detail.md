## 1. カレンダービュー（`/events`, `events#index`）

### 1.1 レイアウト構成

画面は `data-controller="calendar"` を持つ1枚の `div`（`app/views/events/index.html.erb`）と、その下に空の `turbo_frame_tag "modal"` のみで構成される。カレンダー本体は Stimulus コントローラ `calendar_controller.js` が FullCalendar 6.1.15（`dayGridMonth`）を `#calendar-root` にマウントして描画する。

```
┌───────────────────────────────────────────────────────────┐
│ 5月 2026                              [today] [‹] [›]      │ ← ヘッダー行（fc-toolbar）
├───────┬───────┬───────┬───────┬───────┬───────┬───────────┤
│  日   │  月   │  火   │  水   │  木   │  金   │    土      │ ← 曜日ヘッダー
├───────┼───────┼───────┼───────┼───────┼───────┼───────────┤
│ 26    │ 27    │ 28    │昭和の日29│ 30 │  1    │    2       │
│ (前月/次月は淡色表示)                                        │
├───────┼───────┼───────┼───────┼───────┼───────┼───────────┤
│ ...   日付セルが6週分並ぶ。各セルにイベントバーを最大表示件数まで積み上げ  │
└───────────────────────────────────────────────────────────┘
```

### 1.2 ヘッダー領域

| 要素 | 内容 | 実装 |
|---|---|---|
| タイトル | `<span class="calendar-title-month">◯月</span><span class="calendar-title-year">YYYY</span>`。太字・大きめフォントで左寄せ | `calendar_controller.js` の `datesSet` で `.fc-toolbar-title` の中身をJSから毎回書き換え |
| リリースノートアイコン `ⓘ` | タイトル文字列の直後に動的追加されるボタン。クリックで `alert()` によりメッセージ表示（例: `version 1.1:リアルタイム更新追加`）。表示期限は `data-calendar-release-note-until-value`（例 `2026-05-19`）を過ぎると生成されなくなる | `calendar_controller.js` `datesSet` 内、`releaseNoteUntilValue`/`releaseNoteMessageValue` |
| `today` ボタン | グレー角丸ボタン、当月に戻る | FullCalendar 標準ツールバーボタン |
| `‹` `›` ボタン | 濃紺の角丸ボタン2つ、前月・翌月へ移動 | FullCalendar 標準ツールバーボタン（`prev`/`next`） |

ヘッダーの `today`/`‹`/`›` は画面右上にまとめて配置され、タイトルは左上固定。

### 1.3 曜日ヘッダー・日付セル

| 項目 | 仕様 |
|---|---|
| 週の開始 | 日曜始まり（日・月・火・水・木・金・土） |
| 曜日文字色 | 日曜＝赤、土曜＝青、平日＝黒（`.fc-col-header` にクラス付与） |
| 日付の数字 | セル右上に表示。「日」という単位文字は削除し数字のみ表示（`dayCellContent`） |
| 祝日 | `holiday_jp` gem でその年〜翌年分を取得しRailsからJSONで渡す（`data-calendar-holidays-value`）。該当日には日付数字の右横に祝日名（例:「昭和の日」「みどりの日」）を小さく表示し、日付数字を赤字化、セル背景を薄赤（`is-holiday` クラス）にする（`dayCellDidMount`） |
| 土曜のセル背景 | 薄い青（FullCalendar標準の週末スタイルをTailwindでカスタム） |
| 前月・翌月の日付 | セル全体が淡色（グレーアウト）表示。祝日名や当該月の予定は表示されるが数字は薄い |
| 今日のセル | 日付数字が青字で強調（スクリーンショットの `9`, `16`, `23`, `30` などの土曜列と同系色だが、当日セル自体には強い枠線等の追加装飾はなし） |

### 1.4 イベントバー表示

| 項目 | 仕様 |
|---|---|
| 表示形式 | `eventDisplay: "block"` により、終日バー状の矩形で表示（時刻ラベルなし、タイトルのみ） |
| 色 | 単色の青（FullCalendar既定色。イベント種別による色分けなし） |
| 複数日イベント | 開始日から終了日まで横に連結したバーとして描画（例: 「よてい」5/12〜5/15、「セコマ」5/19〜5/20） |
| 1日に複数件 | セル内に縦に積み上げ表示。表示しきれない分は `dayMaxEvents: true` により `+N more` リンクに集約（例: `+2 more`） |
| `+N more` クリック | FullCalendar標準のポップオーバーで残りのイベント一覧を表示（ポップオーバー内イベントクリック時は先に `.fc-popover-close` を自動クリックしてから遷移） |
| クリック時の遷移先 | イベントの `url`（`/events/:id/edit`）を `#modal` という `id` を持つ turbo-frame の `src` に設定し、モーダルとして開く（`eventClick`） |

### 1.5 日付セルクリック時の挙動

- イベントが無い日付部分をクリックすると `dateClick` が発火し、`/events/new?date=YYYY-MM-DD` を `#modal` フレームの `src` にセットして新規作成モーダルを開く。
- `date` クエリはその日の 10:00 をデフォルト開始時刻として渡す（`events#new` 側の実装、[business_logic.md](business_logic.md) 10番）。

### 1.6 リアルタイム反映

- `turbo_stream_from "events_user_#{current_user.id}"` を購読し、同一ユーザーの作成・更新・削除がサーバー側からブロードキャストされる。
- `turbo:before-stream-render` イベントを `calendar_controller#refresh` で横取りし、ターゲットが `calendar-stream-container` の場合は既定のStream描画をキャンセルして代わりに `calendar.refetchEvents()` を呼ぶ（＝ページ描画を伴わずカレンダーのイベント一覧のみ再取得）。

## 2. 予約作成／変更モーダル（`/events/new`, `/events/:id/edit`）

`new.html.erb` と `edit.html.erb` はどちらも共通パーシャル `_modal_form.html.erb` を呼び出す薄いラッパーであり、`title`（見出し文言）と `show_delete`（削除ボタン表示可否）だけが異なる。

| 呼び出し元 | title | show_delete |
|---|---|---|
| `new.html.erb` | `"予約登録"` | `false` |
| `edit.html.erb` | `"予約変更"` | `true` |

### 2.1 全体レイアウト

```
┌──────────────────────────────────────────┐
│ 予約変更                              ×   │ ← ヘッダー（bg-gray-50、下線あり）
├──────────────────────────────────────────┤
│ タイトル                                   │
│ [よてい__________________________]        │
│                                            │
│ 開始日時                                   │
│ [2026/05/12, 10:00            🗓]         │
│                                            │
│ ●== 終了日時を指定する                     │ ← トグルスイッチ（ON時は青、OFF時は灰）
│ [2026/05/15, 11:00            🗓]  ← ON時のみ表示
│                                            │
│ メモ                                       │
│ [予定です                              ]   │
│ [                                       ]  │
│ [                                       ]  │
│                                            │
├──────────────────────────────────────────┤
│ 🗑                    [×]  [✓ 保存]        │ ← フッター（削除・キャンセル・保存）
└──────────────────────────────────────────┘
```

- モーダルは画面中央固定、背景は半透明の黒オーバーレイ（`bg-black bg-opacity-50`）でクリックしても閉じない（背景クリックでの閉じる導線は実装されていない。閉じるには右上×かフッターの×リンクを使う）。
- カード自体は白背景・角丸（`rounded-xl`）・影付き、開閉時にフェードイン系アニメーションが付与される（`animate-backdrop` / `animate-modal-card`）。
- 本文エリアが画面高を超える場合は `overflow-y-auto` でスクロール、ヘッダー・フッターは固定。

### 2.2 ヘッダー

| 要素 | 内容 |
|---|---|
| 見出し | 太字・やや大きめ。「予約登録」または「予約変更」 |
| 閉じるボタン（×） | 右上、SVGのバツアイコン。リンク先は `events_path(start_date: @event.start_time)` — 編集前後にかかわらずカレンダーへ戻るだけで保存はしない |

### 2.3 フォーム項目

| ラベル | 入力要素 | model属性 | 備考 |
|---|---|---|---|
| タイトル | `text_field` | `title` | placeholder「例：外乗、十勝など」。バリデーションエラー時はラベル右隣にピンク文字でエラーメッセージ（例：必須・50文字以内、[models.md](models.md)参照） |
| 開始日時 | `datetime_field`（ブラウザネイティブの日時ピッカー、カレンダーアイコン付き） | `start_time` | `data-time-sync-target="start"`、`change` イベントで後述の自動補完が発火。表示形式はブラウザ依存（スクリーンショットでは `2026/05/12, 10:00`） |
| 終了日時を指定する | トグルスイッチ（`check_box` を `sr-only` で隠し、CSSの `peer-checked` で見た目のスイッチ・下の入力欄の表示/非表示を制御） | `has_end_time` | OFF時は下の日時入力欄ごと非表示（DOMには存在するが `hidden`）。ONにすると入力欄が下にスライド表示される |
| （終了日時入力欄） | `datetime_field` | `end_time` | `has_end_time` がONの時のみ表示。`data-time-sync-target="end"`。バリデーションエラー時は入力欄の上にピンク文字でメッセージ |
| メモ | `text_area`（4行、リサイズ不可） | `content` | placeholder「どんな予定ですか？」 |

エラー表示（フォーム上部）: バリデーションエラーが1件以上ある場合、フォーム最上部にピンク枠のカードが表示され「N箇所ほど、お直しが必要です ₍ᐢ‥ᐢ₎」という文言が出る（`@event.errors.count` を埋め込み）。

### 2.4 終了日時トグルの挙動（スクリーンショット2点の差分）

- **OFF（`18.12.55.png`）**: スイッチが左寄せ・灰色。終了日時入力欄は非表示。モーダルの高さはその分低い。
- **ON（`18.12.41.png`）**: スイッチが右寄せ・青色。直下に終了日時の `datetime_field` が表示される。

### 2.5 開始日時変更時の自動補完（time-sync）

- フォーム全体に `data-controller="time-sync"` が付与されている。
- 開始日時 (`start` ターゲット) の `change` で `time_sync_controller.js#sync` が発火し、終了日時 (`end` ターゲット) の値を「開始日時 + 1時間」に自動セット（`YYYY-MM-DDTHH:mm` 形式）。
- 終了日時トグルがOFF（非表示）でも入力欄自体はDOM上に存在するため値は書き換わる。表示上ユーザーに見えるのはトグルをONにした後。

### 2.6 フッター（アクション行）

| 位置 | 要素 | 表示条件 | 遷移・動作 |
|---|---|---|---|
| 左 | 削除ボタン（ゴミ箱アイコン、赤系背景） | `show_delete: true`（編集画面のみ、新規作成では非表示＝空divのみ） | `DELETE /events/:id`。クリック時に `turbo_confirm` で「本当に削除しますか？」の確認ダイアログ |
| 右（左） | キャンセルボタン（×アイコン、白背景+枠線） | 常に表示 | `events_path(start_date: @event.start_time)` へリンク（保存せずカレンダーへ戻る） |
| 右（右） | 保存ボタン（チェックアイコン＋「保存」、青背景） | 常に表示 | フォーム送信。新規は `POST /events`、編集は `PATCH /events/:id` |

保存・削除いずれも成功後はページ遷移せず、Turbo Streamsでモーダルを閉じつつカレンダー側を再取得する（[business_logic.md](business_logic.md) 11番）。バリデーションエラー時は同モーダルがエラーメッセージ付きで再描画される。

## 3. 画面間の状態遷移まとめ

```
カレンダー(index)
  ├─ 日付セルクリック            → #modal に new をロード（開始10:00デフォルト）
  ├─ イベントバー/+more クリック  → #modal に edit をロード
  └─ (自動) Turbo Streams受信     → refetchEvents() のみ、モーダルは操作しない

モーダル(new/edit)
  ├─ ×(ヘッダー) / キャンセル     → events_path へ遷移（保存なしで閉じる）
  ├─ 保存                        → create/update。成功時はStreamでモーダル空に＋カレンダー更新
  │                                 失敗時は同モーダルをエラー付きで再描画
  └─ 削除（editのみ）             → confirm後 destroy。成功時はStreamでモーダル空に＋カレンダー更新
```
